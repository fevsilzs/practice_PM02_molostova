8 часов. Во второй день практики я узнала и научилась следующему:

Я узнала:

Что такое ER-диаграмма - графическое представление логической структуры базы данных, позволяющее наглядно увидеть таблицы и связи между ними.

Основные компоненты ER-диаграммы: сущности (таблицы), атрибуты (поля), первичный ключ (PK), внешний ключ (FK), связи между таблицами.

Типы связей: один ко многим (1:N), многие ко многим (M:N) - требует промежуточной таблицы, один к одному (1:1) - редко используется.

Я научилась:

Создавать ER-диаграмму в MySQL Workbench: добавлять таблицы, задавать атрибуты с типами данных, отмечать первичные ключи (PK, NN, AI).

Строить связи между таблицами с помощью инструментов 1:N Relationship и N:M Relationship.

Экспортировать ER-диаграмму в PDF для отчётной документации.

Генерировать SQL-скрипт создания базы данных из диаграммы (Forward Engineer).

Выполнять SQL-скрипт на сервере MySQL и проверять создание таблиц.

Импортировать тестовые данные из CSV-файлов через Table Data Import Wizard.

Сохранять все файлы (PDF, SQL, CSV) в репозиторий Git и делать коммит.



8 часов. В третий день практики я узнала и научилась следующему:

Я узнала:

Основные правила построения ER-диаграмм: каждая сущность должна иметь первичный ключ, все атрибуты должны быть атомарными (1НФ).

Как реализуются связи: 1:N - добавлением внешнего ключа в дочернюю таблицу, M:N - через промежуточную таблицу с двумя внешними ключами.

Требования третьей нормальной формы (3НФ) - отсутствие транзитивных зависимостей (неключевые поля не должны зависеть от других неключевых полей).

Рекомендуемые типы данных в MySQL: INT, VARCHAR, DECIMAL, DATE, DATETIME, BOOLEAN, ENUM.

Полную спецификацию ER-диаграммы для своего варианта (таблицы, атрибуты, первичные и внешние ключи, типы связей).

Я научилась:

Создавать ER-диаграмму в MySQL Workbench для своего варианта с нуля.

Добавлять таблицы, задавать атрибуты с правильными типами данных и флагами (PK, NN, AI, UNIQUE).

Строить связи 1:N с помощью инструмента 1:N Relationship - внешний ключ создаётся автоматически.

Строить связи M:N с помощью инструмента N:M Relationship - Workbench сам создаёт промежуточную таблицу.

Проверять внешние ключи и настраивать правила ON DELETE RESTRICT и ON UPDATE CASCADE.

Экспортировать ER-диаграмму в PDF и генерировать SQL-скрипт через Forward Engineer.

Выполнять SQL-скрипт на сервере MySQL и проверять создание таблиц и внешних ключей.

Сохранять диаграмму и скрипт в репозиторий Git.
